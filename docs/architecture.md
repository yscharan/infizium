# Architecture

## Overview

Infizium is a serverless, cloud-native application on AWS. The frontend is a Next.js app hosted on Amplify. All business logic runs in Lambda functions behind API Gateway. Data is stored in Aurora PostgreSQL Serverless v2. Authentication is handled by Cognito. WhatsApp is the primary outbound communication channel.

## Services

### Frontend — Next.js on AWS Amplify
- Server-side rendering for authenticated pages.
- Static generation for public pages (landing, login).
- Amplify handles deployments, custom domains, and environment variables.
- No native mobile app in MVP — the web app is mobile-responsive and installable as a PWA.

### API — API Gateway + Lambda
- One API Gateway (REST) with resource-based routing.
- Lambda functions are grouped by domain: `attendance`, `homework`, `announcements`, `forms`, `profiles`, `notifications`.
- Each Lambda is a thin handler. Business logic lives in shared modules inside the Lambda package.
- Lambda is deployed per environment (dev, staging, prod) using environment variables for config.

### Database — Aurora PostgreSQL Serverless v2
- Serverless v2 scales to zero when idle — low cost for a school platform with predictable low-traffic hours.
- Connection pooling via RDS Proxy to avoid Lambda connection exhaustion.
- Schema migrations managed with a migration tool run from CI (e.g., Flyway or db-migrate).
- One database per environment.

### Auth — Amazon Cognito
- Separate user pools per role: Student, Parent, Teacher, SchoolAdmin.
- JWT tokens validated at the API Gateway level using a Cognito authorizer.
- Parents are identified by their WhatsApp phone number. Cognito phone auth + OTP.
- Teachers and admins use email + password.

### Storage — Amazon S3
- Profile photos.
- Attachment storage for future modules (not used in MVP).
- Files are private by default; accessed via pre-signed URLs generated by Lambda.

### WhatsApp — WhatsApp Business API
- Infizium uses an official WhatsApp Business Solution Provider (BSP) account.
- Outbound messages are sent from Lambda via the BSP HTTP API.
- Inbound messages (parent replies) are delivered to a webhook Lambda via the BSP callback URL.
- Message templates are pre-approved by Meta for transactional notifications (attendance, homework, forms).
- Free-form replies from parents trigger session messages handled by the webhook Lambda.

## Data Model (Core Tables)

```
schools
  id, name, address, district, state, created_at

users
  id, cognito_sub, role (student|parent|teacher|admin), school_id, created_at

student_profiles
  user_id, name, class, section, roll_number, dob, photo_s3_key

parent_profiles
  user_id, name, whatsapp_number, relationship, linked_student_ids[]

  id, student_id, date, period, status (present|absent|late), reason, marked_by, created_at

  id, teacher_id, class, section, subject, description, due_date, created_at

announcements
  id, author_id, target (school|class|section), target_id, title, body, created_at

  id, admin_id, title, fields (JSONB), requires_approval, created_at

  id, form_id, student_id, parent_id, response (JSONB), approved, responded_at

notifications
  id, user_id, channel (whatsapp), type, payload (JSONB), sent_at, delivered_at
```

## Cost Controls
- Aurora Serverless v2 scales to 0 ACUs when idle (school nights, weekends).
- Lambda pricing is per-invocation — no idle cost.
- API Gateway caching is off in dev, optional in prod for read-heavy endpoints.
- S3 lifecycle rules archive old notification payloads after 90 days.
- CloudWatch log retention set to 30 days across all Lambda functions.

---

## Security
- All data in transit: TLS 1.2+.
- All data at rest: AES-256 (Aurora, S3 default encryption).
- IAM roles follow least-privilege. Lambda roles grant only the specific Aurora, S3, and SES/SNS actions needed.
- Student data is never returned to a parent without a verified parent-student link in the database.
- WhatsApp webhook endpoint validates the BSP HMAC signature before processing any message.
- No PII is logged. Lambda structured logs omit student names, parent phone numbers, and message content.
